In this partial we deal with the information to be inserted in the html header. The goal is that the recommended meta tags are always inserted: robots description og:type og:site_name og:title og:description og:url og:image og:image:width og:image:height twitter:card twitter:site twitter:title twitter:description twitter:image even if they don't have any content. Plus: title canonical If an og or twitter meta tag doesn't have any content we take it from the page: title, description, image. For instance, the metas og:title and twitter:title are inserted only if the page field seo_title is not empty. Since we want to insert always these meta tags, here we have to deal with the fields that are empty and use the page's fields instead. The optional meta tags: author keywords will be inserted only if they have any content. These are handled by the extension seo, but it only inserts a meta tag if the related field has content. With this partial, the extension yoast_seo, if installed, is only used for the analysis shown in the BE. {images -> v:iterator.first() -> f:variable(name: 'image')}
